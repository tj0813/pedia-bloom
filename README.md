# Pedia Bloom — Jelajah Nusantara 🇮🇩

A bilingual (Indonesian + English), installable, **fully offline** PWA encyclopedia about Indonesia for children ages 5–13 — assembled from 102 Google Stitch screen designs into one navigable app.

- **Design:** "Premium 3D Storybook" system (`DESIGN.md`) — leaf-green / sky-blue / warm-orange on a warm cream canvas, Quicksand throughout, tactile squishy buttons, soft cardstock shadows.
- **Bilingual everywhere:** every heading and topic shows Indonesian primary + an English subtitle (fresh-teal), per `DESIGN.md`.
- **Self-contained:** no CDNs. Tailwind is compiled to static CSS, fonts are self-hosted, the Material Symbols icon font is instanced down to ~490 KB. The app makes **zero external requests** and works offline on first load.
- **Mascot:** the explorer girl + smart owl. **Language:** Bahasa Indonesia + English.

---

## Run it

**A. Double-click (quickest)** — open `index.html` in Chrome/Edge. Works because the screen bundle and i18n load via `<script src>` (no `fetch`). The service worker stays off on `file://`, so no offline caching, but everything else works.

**B. Local server (full PWA — recommended)** — enables the service worker, offline mode, and "Install app":
```powershell
python -m http.server 8765
# then open http://127.0.0.1:8765/index.html
```

## Build pipeline

Everything ships pre-built, so deployment needs no build step. To regenerate artifacts:

```powershell
npm run build          # full rebuild: bundle -> fonts -> css -> icons
# or individually:
npm run build:bundle   # python build.py        -> app/screens-data.js (+ localizes images)
npm run build:fonts    # python build_assets.py  -> app/assets/fonts/   (needs requirements-dev.txt)
npm run build:css      # tailwindcss             -> app/assets/styles.css
npm run build:icons    # node tests/gen-icons.mjs -> app/assets/icons/
```

Node deps: `npm install` (Tailwind + Playwright, dev-only). Python deps for font self-hosting: `pip install -r requirements-dev.txt` (fonttools + brotli).

## How it's built

Stitch exports each screen as a **standalone HTML island** that re-declares the whole design system and a dead bottom nav. "Making the app" was an *integration* problem.

```
stitch_pedia_bloom/
├── index.html                 # PWA shell: top bar + persistent bottom nav + router mount
├── src/styles.src.css         # Tailwind source + design tokens (CSS vars) + storybook components
├── tailwind.config.js         # static Tailwind config (token colors via CSS vars, safelist)
├── app/
│   ├── app.js                 # hash router, native screens, engagement tracking, bilingual UI
│   ├── screens-data.js        # 102 screen fragments bundled (generated — do not edit)
│   ├── i18n/topics.js         # 102 curated ID→EN titles + subtitles (drives bilingual cards)
│   └── assets/
│       ├── styles.css         # compiled Tailwind (generated)
│       ├── fonts/             # self-hosted Quicksand + instanced Material Symbols (generated)
│       ├── icons/             # PWA PNG icons incl. maskable (generated)
│       ├── img/               # localized topic images (generated by build.py)
│       └── icon.svg, placeholder.svg
├── manifest.webmanifest       # PWA manifest (PNG + maskable icons)
├── sw.js                      # service worker v4 (precache shell, SWR media, offline nav fallback)
├── build.py                   # regenerates screens-data.js + localizes/sanitizes images
├── build_assets.py            # self-hosts + instances fonts
└── tests/                     # e2e.mjs (Playwright, 26 checks) + test_build.py (pytest, 31 tests)
```

### Key architecture decisions
1. **Bundle, don't fetch.** All fragments are inlined into `app/screens-data.js`, loaded via `<script src>`, so the app opens by double-click (`file://` bans `fetch`).
2. **Indirect `eval` for screen scripts.** Each screen's inline JS runs via `(0,eval)(code)`: function declarations leak to global scope (so inline `onclick` keeps working) while `let`/`const` reset per visit.
3. **One token system, two themes.** All 102 fragments + the shell reference Material-3 token names; tokens are CSS variables (`rgb(var(--x) / <alpha>)`) with a `.dark` override, so re-skinning is a value swap and **dark mode is a single class flip** — no per-component `dark:` variants.
4. **Bilingual at runtime.** `app/i18n/topics.js` maps every topic to an English title + subtitle, consumed by the renderer — cards, related, library, map and topic detail are all bilingual without rebuilding the bundle.

## What's in the app

| Area | Route | Notes |
|---|---|---|
| Beranda (Home) | `#/home` | Daily fact, real XP/level/streak, feature cards, recommendations |
| Jelajah (Explore) | `#/explore` | Category grid + quick tiles |
| Library | `#/library`, `#/category/:cat` | All 102 topics, live bilingual search + filter |
| Topic | `#/screen/:id` | Stitch fragment + EN translate-banner + interactions + related |
| Peta (Map) | `#/map`, `#/map/:region` | 7 regions → keyword-matched content |
| Game | `#/games` | Misi & Game category |
| Lencana (Badges) | `#/badges` | 7 badges unlocked by **real** engagement |
| Tanya AI | `#/ai` | BimoBot chat — scripted, child-safe (no live model; see limitations) |
| Mode Orang Tua | `#/parent` → `/dashboard` | Math gate → analytics from tracked usage |
| Pengaturan | `#/settings` | Editable explorer name, dark mode, dyslexia font, larger text |

**Gamification is real, not mocked.** `computeStats()` derives level, XP, streak, coins and badges from `localStorage` visit history — powering the home stats, top-bar chip, and parent dashboard. **No data leaves the device; no backend.**

## Known limitations (honest list)

- **Tanya AI is a scripted, child-safe preview** — keyword-matched answers, no live model input. A real tutor needs a backend + verified-content retrieval (Phase 2).
- **A few Stitch images don't match their topic** (e.g. a "Badak Jawa" card showing a lion) and some titles. These are AI-generation artifacts baked into the *source* exports — curating them is content work, not a code bug.
- **Full body-prose translation is pending.** Headings, titles, navigation and all native UI are bilingual; translating the descriptive paragraphs inside each of the 102 fragments is a separate content milestone (the i18n title map is the seed).
- **Icon font ships all glyphs (~490 KB).** Material Symbols is a ligature variable font; a true per-icon subset isn't possible without losing the FILL toggle, so it's instanced (4 MB → 490 KB) rather than glyph-subset.

## Verified

- **E2E (`npm test`):** 26 Playwright checks — design tokens live, fonts self-hosted, icons render as glyphs, **zero external requests**, all 11 routes + all 102 topic screens render error-free under strict CSP, bilingual present, search/parent-gate work, dark mode flips, manifest installable, **full offline boot**.
- **Unit (`npm run test:py`):** 31 pytest checks on the build pipeline (URL rewriting, sanitising, extraction, category mapping, format sniffing, icon scan).
- Security: strict self-only CSP, no inline secrets, no external origins, `frame-ancestors 'none'`. Deploy headers in `vercel.json`.
